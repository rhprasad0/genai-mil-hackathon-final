# Documentation

This directory now has two jobs:

1. preserve the protected hackathon submission evidence; and
2. describe the Policy Bonfire V1 harness that grows out of it.

Policy Bonfire is a synthetic closed-loop adjudication failure lab. It does not touch real vouchers, real travel records, real payments, real claimants, or official systems. The unsafe agent is the specimen. The harness is the product.

## V1 in one breath

Policy Bonfire runs synthetic packets through low-safeguard adjudication specimens and scores where they produce policy laundering, rubber-stamp theater, decorative human review, audit-log whitewash, unsupported finality, and human-authority boundary collapse.

The first implementation slice is deliberately small and now has a frozen control bundle:

```text
citation manifest
+ scenario card schema
+ 3-card vertical slice
+ 27-run public-safe control pilot
```

The harness has proven it can load scenarios, enforce fake-world boundaries, score outputs, scrub exports, and produce article-ready artifacts. Next work should improve breadth and repeatability without pretending this is a live adjudication system.

## Public-safety rules

All documentation in this repo should remain safe to publish:

- synthetic examples only;
- no secrets, credentials, private notes, raw transcripts, or local logs;
- no real personal, travel, payment, operational, or government-system data;
- no real DTS access, real vouchers, real claimants, real payments, or real fraud accusations;
- no implication of live government deployment or official workflow authority; and
- no product claims that any prototype performs official actions.

The goblins are allowed back in, but only inside the sandbox.
